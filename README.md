# AI Business Analyst (MVP)

**AI-Business Analyst** — консольное приложение для генерации бизнес-требований и имитации AI-анализа проекта по обработке кредитных заявок физических лиц.  

Проект был подготовлен для **хакатона** и демонстрирует, как можно использовать Python для сбора требований, генерации отчета и эмуляции AI-предсказаний.

---

## Функциональность

- Сбор бизнес-требований через консольные вопросы:
  - Цель проекта
  - Проблемы
  - Пользователи
  - Текущий процесс
  - Scope
  - Бизнес-правила
  - KPI
- Генерация HTML-отчета с возможностью конвертации в PDF.
- AI-анализ проекта (имитация) с оценкой автоматизации, рисками и рекомендациями.
- Логирование всех входных данных в папку `logs/`.
- Демонстрационный режим с автозаполнением (для быстрого демо).

## Установка

1. Склонируйте репозиторий:

```bash
git clone <URL_репозитория>
cd ai_project
Убедитесь, что у вас установлен Python 3.11+.

(Опционально) установите необходимые библиотеки:

bash
Копировать код
pip install -r requirements.txt
Для MVP внешние библиотеки не требуются.

Запуск
Обычный режим
bash
Копировать код
python console_mvp.py
Программа будет задавать вопросы по проекту, затем создаст HTML-отчет в папке reports/ и откроет его в браузере.

Демонстрационный режим
bash
Копировать код
python demo_mode.py
Все ответы вводятся автоматически для быстрого показа работы на хакатоне.

Структура проекта
text
Копировать код
ai_project/
├─ console_mvp.py       # Основной скрипт MVP
├─ demo_mode.py         # Автозаполнение для демонстрации
├─ fake_api.py          # Имитация AI-бэкенда
├─ logs/                # Входные данные пользователей
├─ reports/             # Сгенерированные HTML-отчеты
└─ presentations/       # HTML для презентации/демо
AI-Эмуляция
AI-часть проекта имитирует следующие аспекты:

Автоматическая оценка уровня автоматизации.

Потенциал оптимизации процессов.

Риски внедрения.

Рекомендованные этапы внедрения и бизнес-эффект.

Рандомизированная оценка (для демонстрации).

Пример вывода AI:

text
Копировать код
AI-ОЦЕНКА ПРОЕКТА:
Уровень автоматизации: ВЫСОКИЙ
Потенциал оптимизации: 40–60%
Риски внедрения:
- Низкое качество входных данных
- Интеграция с legacy-системами
- Регуляторные ограничения

1. Автоматизация скоринга
